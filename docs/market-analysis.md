## 1. 市場現況概覽

### 全球 LIS 市場規模
- **2024 年市場規模：** 約 USD 37 億
- **預測 CAGR（2024–2030）：** 約 8.2%
- **主要驅動因素：** 數位化醫療轉型、精準醫學需求、遠端醫療普及

### 台灣市場特性
- 全民健保制度下，醫院資訊化程度高，但老舊系統更換週期長（10–15 年）
- 中小型診所和獨立檢驗所數位化需求旺盛，缺乏符合預算的現代化 LIS 解決方案
- 醫療 IT 採購流程保守，Demo 展示與使用者體驗是突破關鍵

## 2. 競爭格局分析

### 主要競爭者

| 競爭者 | 定位 | 介面風格 | 弱點 |
|--------|------|----------|------|
| 訊聯生技 LIS | 大型醫院級 | 功能導向、複雜 | UI 老舊、學習曲線陡峭 |
| 漢鼎資訊 MedLIS | 中型醫院 | 傳統 Windows 風格 | 缺乏現代 UX，移動端差 |
| 精誠健康 | 綜合醫院 IT | 企業風格 | 定制化費用高 |
| 國際品牌（Sunquest, Cerner） | 大型教學醫院 | 英文介面為主 | 本地化不足、授權費昂貴 |
| SaaS 新創 | 小診所 | 簡潔但功能薄 | 缺乏完整檢驗所功能 |

### 市場空白（Blue Ocean）
**現代化設計 × 完整功能 × 合理成本** — 這個交叉點目前無人佔據。

## 3. LISA 獨特賣點（USP）

### 核心差異化：Muji Style 設計哲學

> **「醫療 IT 的第一個日式極簡系統」**

傳統醫療 IT 系統設計邏輯是「功能優先，美感其次」。LISA 顛覆這個邏輯：

| 維度 | 傳統 LIS | LISA |
|------|----------|------|
| 設計哲學 | 資訊堆疊、功能展示 | 減法美學、專注任務 |
| 色彩系統 | 高對比、藍白灰 | 自然色系、米白、暖灰 |
| 字體排版 | 系統預設字型 | 精選字型層次系統 |
| 空白運用 | 盡量填滿空間 | 呼吸感設計，留白有意義 |
| 操作反饋 | 彈窗提示 | 優雅的微互動 |
| 學習曲線 | 需要培訓 | 直覺操作，30 分鐘上手 |

### Muji Style 帶來的商業價值
1. **降低培訓成本** — 介面直覺，人員流動時不需長期訓練
2. **減少操作錯誤** — 清晰的視覺層次降低人為失誤
3. **提升工作滿意度** — 美好的工作環境降低員工離職率
4. **品牌形象提升** — 現代化介面傳遞機構專業度

## 4. 目標客戶群分析

### 優先目標（Primary）

#### A. 中型私人醫院（50–200 床）
- **痛點：** 現有系統老舊，難以招募年輕醫檢師
- **決策者：** 資訊長（CIO）、院長
- **預算：** 年費 NTD 50–150 萬
- **切入點：** 強調人員招募競爭力與操作效率

#### B. 獨立醫學檢驗所
- **痛點：** 大系統太貴、Excel 管理有風險
- **決策者：** 所長、行政主任
- **預算：** 年費 NTD 20–60 萬
- **切入點：** ROI 明確，快速部署，合規需求

### 次要目標（Secondary）

#### C. 連鎖健檢中心
- **特色：** 客戶體驗導向，設計感能直接差異化品牌

#### D. 診所聯盟 / 地區醫療群
- **特色：** 多點管理需求，SaaS 模式最適合

#### E. 醫學院教學醫院（實習環境）
- **特色：** 影響力大，培育未來使用者習慣

## 6. MVP Demo 展示策略

### 展示目標
不是「功能演示」，而是**「體驗銷售」**— 讓潛在客戶感受到介面的美好。

### 三大展示場景

#### 場景一：行政人員開立醫令（2 分鐘）
**故事腳本：** 「護理師 Chen 在早班，需要為 15 位病患開立常規檢驗」
- 展示批量開單的流暢感
- 強調：介面清晰，不需要記憶複雜按鈕位置
- **Wow Moment：** 搜尋病患時的即時過濾動畫

#### 場景二：醫檢師輸入與核發（3 分鐘）
**故事腳本：** 「醫檢師 Lin 同時處理 50 份樣本」
- 展示檢驗值輸入的效率
- 強調：異常值自動標記，視覺清晰
- **Wow Moment：** 報告核發後的優雅完成動畫

#### 場景三：客戶查看報告（1 分鐘）
**故事腳本：** 「患者 Wang 在手機上查看昨日的血液報告」
- 展示報告的可讀性設計
- 強調：一般人也能理解的報告呈現
- **Wow Moment：** 趨勢圖的視覺化呈現

### 展示技巧

| 技巧 | 說明 |
|------|------|
| **先美後功能** | 先讓決策者感受介面美感，再展示功能完整性 |
| **對比展示** | 準備一頁傳統 LIS 截圖，與 LISA 並排對比 |
| **真實數據** | 使用貼近客戶的假數據（台灣常見檢驗項目） |
| **角色切換** | 當場切換三個角色視角，展示完整流程 |
| **留白時間** | 不要填滿每一秒，讓客戶有時間感受 |

### Demo 後的跟進策略
1. **即時問卷（30 秒）：** 「哪個畫面讓您印象最深？」
2. **客製化報告：** 附上與客戶現有系統的對比分析
3. **免費試用期：** 提供 30 天沙盒環境，讓醫檢師實際操作
4. **ROI 計算器：** 估算培訓成本節省、操作效率提升的具體金額

## 7. 市場進入策略建議

### Phase 1：Demo 驗證（0–3 個月）
- 目標：獲得 5 個醫療機構的 Demo 機會
- 管道：醫療 IT 展覽、LinkedIn、醫學學會
- KPI：Demo 轉換率 > 40%

### Phase 2：Early Adopter（3–9 個月）
- 目標：簽下 3 個付費試點客戶
- 策略：以較低價格換取案例故事與回饋
- KPI：NPS > 60

### Phase 3：市場擴張（9–18 個月）
- 目標：20 個付費客戶，月均 ARR NTD 300 萬
- 策略：案例故事行銷、口碑推薦計畫

## 8. 風險評估

| 風險 | 影響度 | 緩解策略 |
|------|--------|----------|
| 醫療法規合規複雜 | 高 | 早期引入法規顧問，確保 HL7/FHIR 合規 |
| 採購週期長（6–18 個月） | 中 | 維持現金流，並行多個潛在客戶 |
| 大廠跟進複製設計 | 中 | 建立品牌認知壁壘，持續迭代 |
| 醫療從業者設計偏好保守 | 低 | 以年輕醫檢師為早期倡議者 |

## 結論

LISA 在醫療 LIS 市場中佔據了一個**明確且防禦性強的利基位置**：Muji Style 設計不只是美觀，而是一種完整的產品哲學，能解決醫療 IT 系統最核心的痛點——複雜、難用、讓人疲憊。

**MVP Demo 的成功關鍵不在於功能完整性，而在於讓每一位看過演示的人都說：「這是我用過最美的醫療系統。」**

## 1. Sprint 目標與範圍

### Sprint 目標
完成 LISA 前端 MVP Demo，涵蓋三大核心功能模組，使用 mock data 模擬完整業務流程，可進行線上展示。

### 範圍
| 模組 | 目標使用者 | 核心功能 |
|------|-----------|---------|
| 醫令開單 | Administrative Staff | 開立檢驗醫令、選擇檢驗項目、填寫病患資訊、醫令清單管理 |
| 檢驗值輸入與報告核發 | Medical Technologist | 輸入檢驗數值、異常值標記、報告核發流程（審核/簽核）、工作清單 |
| 報告查看 | Client | 查看檢驗報告、報告列表與搜尋、報告詳細內容與歷史趨勢 |

### 不在範圍（Out of Scope）
- 後端 API 整合（使用 mock data）
- 使用者認證/授權（僅做 UI 切換）
- 多語系支援
- 列印功能

## 6. 風險與注意事項

| 風險 | 影響 | 緩解措施 |
|------|------|---------|
| 檢驗項目資料模型複雜度高 | 開發延遲 | Marcus 提前定義完整 Schema，預留緩衝 |
